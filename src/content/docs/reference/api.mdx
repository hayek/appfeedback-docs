The reference generators (DocC via `swift-docc-plugin`, Dokka HTML, TypeDoc) are wired into CI to emit static HTML to the sub-paths above. Until that pipeline is published, browse the typed source directly:

- Swift — [`AppFeedbackSDK`](https://github.com/hayek/AppFeedbackSDK) (`Sources/AppFeedbackCore`, `Sources/AppFeedbackUI`)
- Kotlin — [`appfeedback-android`](https://github.com/hayek/appfeedback-android) (`com.appfeedback.core`)
- TypeScript — [`appfeedback-web`](https://github.com/hayek/appfeedback-web) (`packages/*/src`)
:::

The guides cover the common surface — `FeedbackClient` / `FeedbackReport` / `FeedbackTransport`, the formatter & parser, the web transports and relay handler, and the widget. The generated references add every public type and member.
